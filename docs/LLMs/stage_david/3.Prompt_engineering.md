# Prompt Engineering for Large Language Models

`Prompt Engineering` is the discipline of designing inputs (prompts) that guide Large Language Models (LLMs) to 
produce accurate, useful, consistent, and structured outputs.

Unlike traditional programming, where developers write explicit instructions, prompt engineering involves 
communicating intentions to a probabilistic language model.

A well-designed prompt can dramatically improve:

- Accuracy
- Reliability
- Reasoning quality
- Output format
- Hallucination reduction
- Task completion

The learning objectives of this course:

- Understand how LLMs interpret prompts 
- Design effective prompts for different tasks 
- Use zero-shot, one-shot, and few-shot prompting 
- Apply reasoning techniques 
- Generate structured outputs 
- Build prompts for RAG systems 
- Create prompts for AI agents 
- Evaluate and improve prompt quality

## 1. What is a Prompt?

A prompt is the input provided to an LLM. For example, `What is the capital of France?` can be a simple prompt to model,
the model receives the prompt and predicts the most likely continuation.

### 1.1 Why prompt is important?

Consider the following prompts `Tell me about Python.`, the ambiguities are:  
- Programming language or Snake species?
- Beginner or expert level? 
- Short or detailed?

A better prompt:
```text
Explain the Python programming language to a first-year computer science student.
Include:
- History
- Main features
- Typical applications
- Simple code examples
```

You can notice in this prompt, we specify:
- Context/task: Explain the Python programming language
- Audience : a first-year computer science student
- Scope: history of python, etc.

## 2. Anatomy of a Good Prompt

A high-quality prompt often contains:
- Context: Background information 
- Task: What to do
- Constraints: Rules to follow
- Output Format: Desired structure

For example:

```text
You are a university professor teaching databases.

Explain database normalization to undergraduate students.

Requirements:
- Use simple language
- Give practical examples
- Explain 1NF, 2NF, and 3NF

Output format:
1. Definition
2. Examples
3. Summary
```


## 3. Zero-Shot Prompting


`Zero-Shot Prompting` means the model receives instructions(in prompt) without examples.

For example,

```text
Classify the sentiment of this sentence:

"I love this course."

Answer only:
Positive
Negative
Neutral

# expected output
Positive
```

Advantages
- Simple
- Fast
- Minimal prompt length

Disadvantages
- Less reliable for complex tasks 
- Greater output variability

## 4. One-Shot Prompting

Provide a single example before asking the task.

```text
Example:

Sentence: I love this movie.
Sentiment: Positive

Now classify:

Sentence: This product is terrible.
Sentiment:

# expected output
Negative
```

The example teaches the model:

- Expected task
- Desired format
- Output style

## 5. Few-Shot Prompting

Provide multiple examples.

```text
Sentence: I love this course.
Sentiment: Positive

Sentence: This is awful.
Sentiment: Negative

Sentence: The weather is okay.
Sentiment: Neutral

Sentence: The food was excellent.
Sentiment:
```

Few-shot prompting often improves:

- Classification accuracy
- Consistency
- Formatting

## 6. Role Prompting

Assign a role to the model.

```text
You are a cybersecurity consultant.

Explain SQL injection to a software engineering team.
```

The model activates patterns associated with:

- Expertise
- Terminology
- Communication style

## 7. Chain-of-Thought Prompting

Encourage step-by-step reasoning.

For example

```text
A store sells apples for €2 each.

Think step by step and explain your reasoning.

If I buy 5 apples, how much do I pay?
```

Complex problems become multistep simple problem.
