## 2 Evolution of Natural Language Processing (NLP)

**What is NLP?**

`Natural Language Processing (NLP)` is the branch of AI focused on enabling computers to "understand", generate, and manipulate human language.

Main Categories:
- Rule-Based NLP
- Statistical NLP

### 2.1 Early NLP: Rule-Based Systems

Early NLP systems relied on `manually written rules`.

For Example:
```text
IF sentence contains "hello"
THEN respond "Hi!"

```
Advantages:
- Easy to understand
- Deterministic behavior
- Good for narrow domains

Limitations :
- Hard to scale
- Brittle rules 
- Cannot generalize 
- Difficult to maintain

`ELIZA (1966) was one of the earliest chatbots`, it simulated a psychotherapist using pattern matching.

For example:
```text
User: I feel sad
ELIZA: Why do you feel sad?
```

> ELIZA did not understand language
> It only matched patterns
> 

### 2.2 N-Gram Models(Statistical NLP)

`Rule-based systems` failed to scale to real language complexity.

Statistical Language Models `predict the probability of words`.

`N-Gram Models` predicts the next word using previous words.

```text
P("learning" | "I love machine")= 0.98
P("gun" | "I love machine")= 0.88
```

Problems of N-Gram Models
- Data sparsity (user input only uses a tiny fraction of the entire vocabulary)
- Poor long-range memory
- Huge storage requirements

### 2.3 Word Embedding

Before 2012, `Traditional Word Representation` represented words as one-hot vectors. For example "king=[1,0,...,0]", "queen=[0,1,0,...,0]". 

The problem with this approach:
- large vector : 50000 words means the vector has 50000 dimensions
- no semantic relations between words

In 2013, `Word2Vec` introduced dense vector representations. For example, "king=[0.23, -0.45, 0.67, ..., 0.12]"
Main improvement is `Cosine similarity between vectors reflects semantic closeness`. For example, `vector("king") - vector("man") + vector("woman") ≈ vector("queen")`

> Word embedding is the fondation of modern LLMs.
> 
> 
### 2.4 Deep learning models

Deep learning allowed models to learn word representations automatically.

Key architectures:
- RNN: RNNs process text sequences step by step, but they struggle with long-term dependencies.
- LSTM: LSTMs improved context memory handling.

They had major problems:
- `Sequential processing`: Very slow training (cannot parallelize well)
- `Vanishing/Exploding gradients`: Poor at handling long sequences
- Hard to capture long-range dependencies

### 2.5 Attention Mechanism

The core idea of attention mechanism is `Instead of remembering everything in one vector, the model dynamically focuses on important words.`

For example, in a sentence like "The animal didn't cross the street because it was tired.", the attention mechanism helps
determine "it refers to 'the animal'"

### 2.6 The transformer architecture

The original Transformer is an `Encoder-Decoder model`, the workflow looks like :
```text
Input Text → Encoder Stack → Decoder Stack → Output Text
```
- `Encoder`: Understands the input (reads the entire sequence at once)
- `Decoder`: Generates the output (auto-regressively)

> Modern LLMs (e.g. GPT, LLaMA, etc.) are decoder-only transformers because they are optimized for text generations.
> 

The below figure shows a general architecture of transformer:

![transformer_architecture.webp](./imgs/transformer_architecture.webp)

```text
Tokens → Embedding + Positional Encoding
                ↓
        [N × Encoder Layers]
                ↓
        [N × Decoder Layers]
                ↓
          Linear + Softmax
                ↓
             Output Tokens
```

The advantage of transformer architecture:
- Parallel processing 
- Long-context understanding 
- Efficient scaling 
- Strong reasoning capabilities

#### The encoder stack

The encoder stack can have N times of the encoder block, each block can have the below layers:
1. Multi-Head Self-Attention 
2. Add & Norm 
3. Feed-Forward Network 
4. Add & Norm

#### The decoder stack

The decoder stack can have N times of the decoder block, and each block can have the below layers:
1. Masked Multi-Head Self-Attention 
2. Add & Norm 
3. Multi-Head Cross-Attention (to encoder)
4. Add & Norm 
5. Feed-Forward Network 
6. Add & Norm

#### Multi-Head Self-Attention (The Core Innovation)

`Multi-Head Self-Attention` is the most important part of the Transformer. It's responsible for:
- Information exchange between tokens
- Context understanding
- Relationship modeling

**Scaled Dot-Product Attention**:

For a sequence of vectors $  X  $, we create three matrices:

Query (Q)
Key (K)
Value (V)

Then compute attention as:
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
Why divide by $  \sqrt{d_k}  $? To prevent the dot products from becoming too large (which would make softmax too peaky).

> `Self-Attention` means the model attends to all positions in the input relative to each other.
> 
> `Multi head`(typically 8 or 16 heads) means we run multiple attention mechanisms in parallel.
> Each head learns to attend to different types of relationships (syntax, semantics, coreference, etc.).
> Outputs are concatenated and projected.
> 
> `Masked` means the model only looks at past tokens 

#### Add & Norm (Residual Connections + Layer Normalization)

After each major sub-layer (attention or feed-forward):

- Residual connection: Add the input back to the output (x + sublayer(x))
- Layer Normalization: Stabilizes training

This helps with gradient flow in very deep networks (Transformers can have 32–128+ layers).


####  Feed-Forward Network (FFN)


The FFN is a `simple two-layer fully connected network` applied to `each position(token) independently`:
$$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$

FFN usually expand dimension (e.g., model dimension=4096, FFN dimension=16384) at the start, then project back to normal.
The larger hidden layer allows:
- Richer feature combinations
- More expressive representations
- Complex nonlinear transformations

This is similar to `Hidden layers in classical neural networks`.

The FFN performs:
- Nonlinear transformation
- Feature extraction
- Representation enrichment
- Computation on each token independently

Without FFN:
- The transformer would behave almost like repeated weighted averaging
- Expressiveness would be severely limited

> Imagine a meeting, the `self attention` is like `People(token) listen to each other to exchange information.`
> `FFN` is like `each person(token) privately thinks and updates understanding of other person(token)`. Without
> FFN, Every person only repeats information, it will be no deeper understanding emerges.

#### Encoder vs Decoder Differences

|Component|Encoder|Decoder|
|---------|--------|-------|
|Self-Attention |Full (bidirectional)| Masked (only looks at past tokens)|
|Cross-Attention|Not present|Attends to Encoder output|
|Purpose| Understanding input|Generating output|


> In Decoder-only models (used in GPT-style LLMs), there is only `masked self-attention`(the model can only attend to 
> previous tokens during generation).
> 
> 

## 3. Large Language Model (LLM)

A Large Language Model is a `deep neural network` based on `transformer architecture` trained on massive 
amounts of text datasets collected from books, websites, scientific papers, code repositories, and many other sources.

The main objective of an LLM is surprisingly simple, it predicts the next token (word or subword) in a sequence. 

Key concepts in LLM

1. Tokenization: text -> token(numeric value)
2. Embeddings: token -> dense vector
3. Transformer Architecture: use a mechanism called `self-attention`, which allows the model to dynamically focus on important parts of the input sequence.

### 3.1 How LLMs Learn

LLMs are trained using self-supervised learning.

During training:
1. A sentence is provided to the model 
2. One token is hidden 
3. The model predicts the missing token 
4. The prediction error is computed 
5. Model parameters are updated

This process is repeated across billions or trillions of tokens.

Over time, the model learns:

- Grammar
- Syntax
- Facts
- Reasoning patterns
- Programming languages
- Human conversation structures

### 3.2 types of LLM architectures


#### 3.2.1 Decoder-Only Models

`Decoder-Only models` generate text `autoregressively`, meaning they predict one token at a time.
Examples:

- GPT family
- Llama
- Mistral


Common applications:

- Chatbots
- Code generation
- Content generation

#### 3.2.2 Encoder-Decoder Models

Encoder-Decoder Models first encode the input text, then generate transformed output text.

Examples:

- T5
- BART

Common applications:

- Translation
- Summarization
- Question answering
### 3.3 LLM Generation parameters

When a Large Language Model (LLM) generates text, it does not simply choose words randomly.
At every step, the model predicts probabilities for the next possible token.

Generation parameters control:

- Creativity
- Randomness
- Diversity
- Coherence
- Determinism

The most important parameters are:

- Temperature 
- Top-k sampling
- Top-p

Practical Recommendations

| Use Case	            | Temperature	 | Top-p     |
|----------------------|--------------|-----------|
| Coding assistant	    | 0.1–0.3      | 	0.8      |
| Scientific assistant | 	0.2–0.5     | 	0.8–0.9  |
| General chatbot      | 	0.7	        | 0.9       |
| Story generation     | 	1.0–1.3     | 	0.95     |
| Brainstorming        | 	1.2         | 	0.95–1.0 |

#### 3.3.1 Temperature

Temperature controls randomness in token selection. 

With a low temperature = 0.1, the behavior of the model:
- Very deterministic
- Safer 
- More repetitive 
- Less creative

With a high temperature = 1.5, behavior of the model:
- More randomness
- More creativity
- More diversity
- More hallucinations possible

Typical Temperature Values

| Temperature | Behavior               |
|-------------|------------------------|
| 0	          | Greedy deterministic   |
| 0.2–0.5     | Focused and reliable   |
| 0.7–1.0     | Balanced creativity    |
| 1.2+        | Highly creative/random |


#### 3.3.2 Top-k Sampling

`Top-k` limits token selection to the k most probable tokens. For example, top_k = 3, the model only keeps the first 
three token with the highest probability. All other tokens are discarded

Top-k prevents:

- Extremely unlikely tokens
- Nonsensical outputs
- Random noise

while still allowing diversity.

#### 3.3.4 Top-p Sampling (Nucleus Sampling)

`Top-p` dynamically selects the smallest set of tokens whose cumulative probability exceeds p. For example,
- we set top-p = 0.80, 
- the first token with proba of `0.72`, Cumulative Probability = 0.72 < 0.8(p value), continue to 2nd token
- 2nd token with proba of `0.11`, Cumulative Probability = 0.72+0.11 = 0.83 > 0.8, stop
- all other tokens are discarded

Why Top-p is Better compare to top-k:

- Dynamic
- Adapts to uncertainty
- Better quality-diversity balance

> modern LLMs often prefer top-p.
> 
Typical Top-p Values:

| top_p    | 	Behavior       |
|----------|-----------------|
| 0.5	     | Conservative    |
| 0.8	     | Balanced        |
| 0.9–0.95 | 	Creative       |
| 1.0      | 	No restriction |

### 3.4 Some useful terms in LLM

- Multimodal models : a model which can treat text, image, audio, and video.
- Fine-tuner a LLM: re-train an LLM, on a more specific dataset
- Zero-shot learning: use the LLM directly on a task
- Few-shot learning: use just a few examples to train the LLM on a task
- Pretext task: the task on which the LLM is trained
- Downstream task: the task associated with the specific use case
- Pretrained: The FM (in this case, LLM) has already been trained; all that remains is to adapt it to our use cases
