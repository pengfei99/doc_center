## 1. Why we need tokenization and embedding

Language models cannot directly understand raw human language text, instead it process:
- Tokens (Numbers)
- Vectors (A list of numbers)
- Matrices

Therefore, text must be converted into numerical representations.

Text processing pipeline

```text
Raw Text
   ↓
Preprocessing
   ↓
Tokenization
   ↓
Vocabulary Mapping
   ↓
Token IDs
   ↓
Embeddings (Vectors)
   ↓
Neural Network Input
```

## 2. Text Preprocessing

The objective of `Text Preprocessing` is to reduce variability and normalize text before tokenization.

Common preprocessing steps:
- `Lowercasing` : "Hello" → "hello"
- `Removing punctuation` : "hello!" → "hello"
- `Unicode normalization` : Standardize characters
- `Removing extra spaces`: "hello world"
- `Handling emojis` :	🙂 -> token
- `Handling URLs` : Replace with special token

For example 

```text
Hello!!! I love LLMs 😊 -> hello i love llms 😊
```

## 3. Vocabulary Construction

A vocabulary is the set of all tokens known by a model.

For example, if a dataset only contains 4 word. The model trained with dataset will have a vocabulary like

| Token  | ID |
|--------|----|
| hello	 | 0  |
| world	 | 1  |
| cat    | 2  |
| dog    | 3  |

> Modern LLM vocabularies size is very large (e.g. GPT-2: 50K, Gemma-3: 256K)
