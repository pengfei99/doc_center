# WebHDFS api 

In this tutorial, we will see how to enable WebHDFS api in a hdfs cluster. Then how to use a WebHDFS client to connect to the
the WebHDFS api.

You can find the official doc [here](https://hadoop.apache.org/docs/r1.0.4/webhdfs.html).

WebHDFS is not a single endpoint system, the client needs to interact with both `NameNode` and `DataNode`.

- NameNode → metadata + redirect
- DataNode → actual data transfer

So the `NameNode config` and `DataNode config` must be perfect.

> Even if the NameNode works, if DataNode HTTP port is unreachable the WebHDFS appears “broken”


The below config example is the minimum config for WebHDFS to work, you can also integrate sasl(https) and kerberos

## 1. Name node config

Edit the `core-site.xml`, you can notice there is no significant change in the `core-site.xml` file.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://spark4-01:8020</value>
  </property>
  <property>
    <name>hadoop.tmp.dir</name>
    <value>file:///home/spark/tmp</value>
  </property>
  <property>
    <name>io.file.buffer.size</name>
    <value>16384</value>
  </property>
</configuration>

```

Edit `hdfs-site.xml`, there are three minimum properties to config:
- dfs.webhdfs.enabled: true to enable webhdfs, false to disable
- dfs.namenode.http-address: specify namenode webhdfs port
- dfs.datanode.http.address: specify datanode webhdfs port

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>2</value>
    </property>

    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///home/spark/hdfs/namenode</value>
    </property>

    <property>
        <name>dfs.namenode.checkpoint.dir</name>
        <value>file:///home/spark/hdfs/secondary</value>
    </property>

    <!-- Enable webhdfs  -->
    <property>
        <name>dfs.webhdfs.enabled</name>
        <value>true</value>
    </property>
    <!-- specify service ports for NameNode  -->
    <property>
        <name>dfs.namenode.http-address</name>
        <value>0.0.0.0:9870</value>
    </property>
    <!-- specify service ports for DataNode  -->
    <property>
        <name>dfs.datanode.http.address</name>
        <value>0.0.0.0:9864</value>
    </property>
</configuration>
```

## 2. Datanode config

The `core-site.xml` file should be exactly looks like the name node.

The `hdfs-site.xml` file should contain three properties
- dfs.datanode.http.address: which port the datanode use for webhdfs
- dfs.client.use.datanode.hostname: force datanode to use hostname, because internal IPs of datanode may not usable
- dfs.datanode.hostname: if the default hostname of the datanode is not reachable by the client, specify a host name which is reachable.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>2</value>
    </property>

    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///home/spark/hdfs/datanode</value>
    </property>

    <!-- Data node http port   -->
    <property>
        <name>dfs.datanode.http.address</name>
        <value>0.0.0.0:9864</value>
    </property>
    
    <property>
        <name>dfs.client.use.datanode.hostname</name>
        <value>true</value>
    </property>
 
   <!-- you need to adapt this value for each data nodes   -->
    <property>
        <name>dfs.datanode.hostname</name>
        <value>spark4-02</value>
    </property>

</configuration>
```

## 3. Test your WebHDFS

```shell
# call namenode
curl -i "http://spark4-01.casdds.casd:9870/webhdfs/v1/?op=LISTSTATUS"

# test redirect behavior
curl -i "http://spark4-01.casdds.casd:9870/webhdfs/v1/test.txt?op=OPEN"

# follow redirect
curl -L "http://spark4-01.casdds.casd:9870/webhdfs/v1/test.txt?op=OPEN"
```

## 4. Use a WebHDFS client (rclone)

`Rclone` is a `cross platform command-line program` to manage files on cloud storage. It is a feature-rich alternative to cloud vendors' 
web storage interfaces. rclone supports Over `70 cloud storage products (e.g. s3, hdfs, etc.).

### Download the tool

You can download the tool from this [page](https://rclone.org/downloads/). For windows, it's quite simple, a single `.exe` will be downloaded and
its ready to go.

### configuration

The configuration for different storage is quite difference. Here we only show how to config hdfs and s3.

You can get the complete list of the supported storage configuration in this [page](https://rclone.org/docs/)

For hdfs, you can check this [page](https://rclone.org/hdfs/)
