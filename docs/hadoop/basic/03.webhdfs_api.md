# hdfs client 

There are three major hdfs clients:
- default hdfs client via RPC
- WebHDFS client via http
- HttpFs client via http

## WebHDFS vs HttpFs

The core difference is the architecture

For `WebHDFS`, the client first talks to NameNode to get the block location, then tals to DataNode directly to get the 
data blocs. As a result, the client must have the ability to talk to all nodes in the cluster

Pros:
- Faster (Direct DataNode transfer)
- No proxy overhead

Cons:
- Complexe network setups: Must handle redirects + client must reach DataNodes
- Client must support SPNEGO correctly

```text
Client
  ↓
NameNode (metadata)
  ↓ (redirect client to data nodes)
DataNode (data transfer)
```

For HttpFS, the client talks to only one `gateway` server, the `gateway` server gathers data blocs and send back to the client.
the client never access/see the datanode. In this mode, the client only needs to talk one server, so if you have 

Pros:
- Single entrypoint: no redict

Cons:
- Slower: Data goes through proxy
- Adds latency + CPU load

```text
Client
  ↓
HttpFS server
  ↓
NameNode + DataNodes (internal)
```

## WebHDFS configuration

You can find the official doc [here](https://hadoop.apache.org/docs/r1.0.4/webhdfs.html).

WebHDFS is not a single endpoint system, the client needs to interact with both `NameNode` and `DataNode`.

- NameNode → metadata + redirect
- DataNode → actual data transfer

So the `NameNode config` and `DataNode config` must be perfect.

> Even if the NameNode works, if DataNode HTTP is unreachable the WebHDFS appears “broken”

### Name node config

Edit `core-site.xml`
```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://spark4-01:8020</value>
  </property>
  <property>
    <name>hadoop.tmp.dir</name>
    <value>file:///home/spark/tmp</value>
  </property>
  <property>
    <name>io.file.buffer.size</name>
    <value>16384</value>
  </property>
</configuration>

```

Edit `hdfs-site.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>2</value>
    </property>

    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:///home/spark/hdfs/namenode</value>
    </property>

    <property>
        <name>dfs.namenode.checkpoint.dir</name>
        <value>file:///home/spark/hdfs/secondary</value>
    </property>

    <!-- Enable webhdfs and specify service ports  -->
    <property>
        <name>dfs.webhdfs.enabled</name>
        <value>true</value>
    </property>

    <property>
        <name>dfs.namenode.http-address</name>
        <value>0.0.0.0:9870</value>
    </property>

    <property>
        <name>dfs.datanode.http.address</name>
        <value>0.0.0.0:9864</value>
    </property>
</configuration>
```

### Datanode config

The `core-site.xml` file should be exactly looks like the name node.

The `hdfs-site.xml` file should contain three properties
- dfs.datanode.http.address: which port the datanode use for webhdfs
- dfs.client.use.datanode.hostname: force datanode to use hostname, because internal IPs of datanode may not usable
- dfs.datanode.hostname: if the default hostname of the datanode is not reachable by the client, specify a host name which is reachable.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>2</value>
    </property>

    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:///home/spark/hdfs/datanode</value>
    </property>

    <!-- Data node http port   -->
    <property>
        <name>dfs.datanode.http.address</name>
        <value>0.0.0.0:9864</value>
    </property>
    
    <property>
        <name>dfs.client.use.datanode.hostname</name>
        <value>true</value>
    </property>
 
   <!-- you need to adapt this value for each data nodes   -->
    <property>
        <name>dfs.datanode.hostname</name>
        <value>spark4-02</value>
    </property>

</configuration>
```

### Test your WebHDFS

```shell
# call namenode
curl -i "http://spark4-01:9870/webhdfs/v1/?op=LISTSTATUS"

# test redirect behavior
curl -i "http://spark4-01:9870/webhdfs/v1/test.txt?op=OPEN"

# follow redirect
curl -L "http://spark4-01:9870/webhdfs/v1/test.txt?op=OPEN"
```
