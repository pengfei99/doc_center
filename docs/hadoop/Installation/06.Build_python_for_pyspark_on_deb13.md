# Build python 3.11 for pyspark on deb13

If you use spark and pyspark 3.5.* or older version in debian 13. you may encounter 


## 1. Install required system dependencies

To build a python 3.11 from source, we need the `development headers for compression, encryption, and database connectivity` 
to ensure PySpark can handle parquet, snappy, and SSL-encrypted HDFS/MinIO streams.

If your server has internet you can just do `sudo apt install`

```shell
sudo apt install build-essential gdb lcov pkg-config libssl-dev zlib1g-dev liblzma-dev libffi-dev libbz2-dev libreadline-dev libsqlite3-dev uuid-dev libncurses-dev libgdbm-dev libreadline-dev libdb5.3-dev tk-dev libnss3-dev
```

If you are in an air-gapped server, you can download the `.deb` file first.

```shell
sudo apt download build-essential gdb lcov pkg-config libssl-dev zlib1g-dev liblzma-dev libffi-dev libbz2-dev libreadline-dev libsqlite3-dev uuid-dev libncurses-dev libgdbm-dev libreadline-dev libdb5.3-dev tk-dev libnss3-dev
```
> you should find all .deb file in the current folder. You can import them into the server.
> 
```shell
sudo dpkg -i *.deb
# If there are missing dependencies, use:
sudo apt-get install -f
```

ps: in debian 11, 12, the package `libncurses-dev` is called `libncurses5-dev`

## Get the python source

You can get the python source from [here](https://www.python.org/downloads/). 

Below is an example for downloading the source
```shell
wget https://www.python.org/ftp/python/3.11.15/Python-3.11.15.tar.xz
tar -xf Python-3.11.15.tar.xz
cd Python-3.11.15
```

## Build the source

```shell
# Configure with optimizations and shared libs
./configure --enable-optimizations \
            --with-lto \
            --enable-shared \
            --prefix=/opt/python311

# Build (using all CPU cores)
make -j $(nproc)

# Install (altinstall prevents overwriting the system Python 3.13)
sudo make altinstall
```

> When you run the config command, there will be a lot of system checks, you will see many yes and some nos. Don't worry

### Errors during build

When you run `make -j $(nproc)`, it will do the actual build. If you see error such as:
- Failed to build these modules: _curses
- Failed to build these modules: _ctypes

You need to check the system dependencies which you installed in `step1`. 

> You should have all the modules correctly build, even though you don't need all of them in your pyspark session.
> 
> 
### Package your python binary

```shell
# after your command `sudo make altinstall`
ls -lah /opt/python311/

# expected output
total 24K
drwxr-xr-x 6 pliu pliu 4.0K Apr 30 10:41 .
drwxr-xr-x 9 root root 4.0K Apr 30 11:15 ..
drwxr-xr-x 2 pliu pliu 4.0K Apr 30 10:44 bin
drwxr-xr-x 3 pliu pliu 4.0K Apr 30 10:41 include
drwxr-xr-x 4 pliu pliu 4.0K Apr 30 10:41 lib
drwxr-xr-x 3 pliu pliu 4.0K Apr 30 10:41 share

# go to the root folder of python311
cd /opt

# create the tar.gz file
sudo tar -czf python311-portable.tar.gz python311
```


## Test your build

As we build the python with the option `--enable-shared`, we need to `link the shared lib` to make the python binary work.

```shell
echo "/opt/python311/lib" | sudo tee /etc/ld.so.conf.d/python311.conf
sudo ldconfig
```
> If you put your python bin in another folder than /opt, you need to change the "/opt/python311/lib" to the correct path

The below command test if your python interpreter contains the essential lib or not

```shell
/opt/python311/bin/python3.11 -c "import sys,sqlite3,bz2,lzma,ctypes,uuid; print('OK')"
```
## config spark

```shell
export PYSPARK_PYTHON=/usr/bin/python3.11
export PYSPARK_DRIVER_PYTHON=/usr/bin/python3.11
```
