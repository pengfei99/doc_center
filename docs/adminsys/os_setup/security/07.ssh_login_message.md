# ssh login message

After you connect to a server via ssh, you often see a default Linux login message.

On Debian, the SSH welcome message can come from several places:

- pre-login banner (/etc/issue.net)
- post-login MOTD (“Message of the Day”)
- dynamic system info generated by PAM scripts

## 2. Disable Debian dynamic MOTD scripts

Debian often generates extra messages from scripts in `/etc/update-motd.d/`

```shell
# List them:
ls -l /etc/update-motd.d/

# Disable a script:
sudo chmod -x /etc/update-motd.d/10-uname

# Or disable all:
sudo chmod -x /etc/update-motd.d/*
```

Then your /etc/motd becomes the only displayed message.

## 3. Configure a pre-login SSH banner

The message in `/etc/issue.net` appears before authentication.


```text
Example:
WARNING: Authorized users only.
All connections are monitored.
```

Edit SSH server config:

```shell
sudo nano /etc/ssh/sshd_config

# add the below line
Banner /etc/issue.net

# Restart SSH:
sudo systemctl restart ssh
```

## 4. User-specific welcome message

Login message for only one user:

```shell
vim ~/.bashrc

# add the below line
echo "Welcome back, $(whoami)"

# Or more advanced:
echo "Today is $(date)"
echo "Server: $(hostname)"
```

## 5. Completely clean login output

To suppress almost everything:

```shell
sudo chmod -x /etc/update-motd.d/*
sudo truncate -s 0 /etc/motd
touch ~/.hushlogin
```
> ~/.hushlogin disables last login and some shell messages.

Typical enterprise setup:

- `/etc/issue.net` -> legal/security warning
- `/etc/motd` -> operational info
- `/etc/update-motd.d/` -> disabled unless system metrics are needed

The Difference between the files

| File                  | When shown    | Typical use             |
|-----------------------|---------------|-------------------------|
| /etc/issue.net	       | Before login	 | Legal warning           |
| /etc/motd	            | After login	  | Welcome banner          |
| /etc/update-motd.d/*	 | After login   | 	Dynamic system info    |
| ~/.bashrc	            | Shell startup | 	Per-user custom output |
