# Use pypi server in debian server

There are two different architectures:
- use Nginx to serve packages as pypi server
- serve packages as local file system

## 1. Set up a pypi server in linux with Nginx

Dans la VM Linux, un point de montage Windows associe le dossier 
web/casd-src6/WEB_RO/repopyt sur Windows au chemin 
/mnt/WEB_RO/REPOPYT dans Linux.

Voir [[datascience:admin_system:linux:mountntfssouslinux|Montage d'un partage CIFS sous Linux avec distribution via Nginx]]

==== 1. Installation des prérequis ====

Installer les utilitaires nécessaires pour gérer les systèmes de fichiers et Python :

<code bash> 
sudo apt update 
sudo apt install -y python3 python3-pip nginx </code>

==== 2. Configuration Nginx pour le serveur PyPI ====

Créer un fichier de configuration Nginx :
<code bash> sudo nano /etc/nginx/sites-available/pip-mirror </code>

Ajouter le contenu suivant :

<code nginx> server { 

   listen 8080; 
   server_name localhost;
   
   location /simple/ {
       alias /mnt/WEB_RO/REPOPYT/web/simple;
       autoindex on;
   }
   
   location /packages/ {
       alias /mnt/WEB_RO/REPOPYT/web/packages;
       autoindex on;
   }
   
   default_type application/octet-stream;

}
</code>

Activer la configuration et tester Nginx :

<code bash> 
sudo ln -s /etc/nginx/sites-available/pip-mirror /etc/nginx/sites-enabled/ 
sudo nginx -t 
sudo systemctl reload nginx </code>

==== 3. Installation de paquets depuis le serveur PyPI local ====

Installer des paquets Python depuis le serveur local :

<code bash> python3 -m pip install --index-url http://localhost:8080/simple numpy --trusted-host localhost </code>


==== 4. Configure pip for all users ====

Based on your linux distribution, the pip config file path may be different.

For `ubuntu`, you can use `/etc/pip.conf.d/00-private-pypi.conf` to put the below conf

```shell
sudo mkdir -p /etc/pip.conf.d 

sudo tee /etc/pip.conf.d/00-private-pypi.conf > /dev/null <<EOF
[global]
index-url = http://localhost:8080/simple
trusted-host = localhost
EOF
```

For `debian 13`, you need to use `/etc/pip.conf`


```shell
sudo touch /etc/pip.conf

sudo tee /etc/pip.conf > /dev/null <<EOF
[global]
index-url = http://localhost:8080/simple
trusted-host = localhost
EOF
```

For more details, Doc details est ici [[http://repo.casd.me:3000/CASD/python_in_CASD/src/branch/main/docs/package-management/pip_configuration.md]]
